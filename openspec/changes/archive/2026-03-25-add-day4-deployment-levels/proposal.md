# Proposal: Add Day 4 部署上线

## Why

平台目前有 20 个关卡，覆盖了基础命令、权限管理、故障排查。Day 4 专注于**前端部署实战**，让用户学习如何将前端项目从构建到上线的完整流程。

用户将扮演**刚完成项目开发的前端工程师**，需要：
- 理解 `npm run build` 构建流程
- 学会 Nginx 配置与服务管理
- 掌握反向代理、日志查看等运维技能

这是真实工作中最常见的场景，填补了"写完代码 → 上线"之间的知识空白。

## What Changes

### 新增内容

| Level | 标题 | 场景故事 | 核心知识点 |
|-------|------|----------|------------|
| 21 | 构建打包 | Vue 项目开发完成，执行构建命令 | npm run build, dist/ |
| 22 | 构建产物 | 查看打包后生成了哪些文件 | ls -lh, 产物结构 |
| 23 | 部署文件 | 将构建产物复制到 Web 目录 | cp -r, /var/www/html |
| 24 | 了解配置 | 查看 Nginx 配置文件结构 | /etc/nginx/nginx.conf |
| 25 | 配置虚拟主机 | 为应用编写专属 Nginx 配置 | server block, location |
| 26 | 检查配置 | 用 nginx -t 检查语法 | nginx -t |
| 27 | 启动服务 | 启动 Nginx，网站上线 | nginx, ps |
| 28 | 测试访问 | 用 curl 测试网站响应 | curl -I, curl -v |
| 29 | 查看日志 | 查看访问日志，了解请求记录 | tail, access.log |
| 30 | 终极挑战 | 配置反向代理，转发 API 请求 | proxy_pass, nginx -s reload |

### 技术实现

1. **Docker 镜像更新**：
   - 安装 nginx
   - 预置一个简单的 Vue 项目
   - 配置必要的目录和权限

2. **后端验证扩展**：
   - 支持 Nginx 配置文件验证
   - 支持 curl 输出验证
   - 支持进程检查

3. **前端关卡扩展**：
   - 添加 Chapter 4 配置
   - 10 个新关卡的数据结构

## Impact

- **用户影响**: 学习真实的前端部署流程，从构建到 Nginx 配置到反向代理
- **技术影响**:
  - Dockerfile: 添加 nginx 和 Vue 示例项目
  - 后端: 扩展验证器，支持 Nginx 相关验证
  - 前端: 扩展 LEVELS 数组，添加 Chapter 4
- **兼容性**: 完全向后兼容，基于现有 20 关扩展到 30 关
